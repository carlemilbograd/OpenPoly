## What is this?

OpenPoly connects your OpenClaw AI agent to a live Polymarket account. The agent reads `SKILL.md` to understand every capability, then calls the right Python script for whatever you ask ‚ÄĒ from a simple portfolio check to running five concurrent trading strategies.

```
You: "Find arbitrage and run the news pipeline in dry-run mode"
       ‚ĒĒ‚ĒÄ‚Ėļ Agent calls correlation_arbitrage.py + news_trader.py
                                        ‚ĒĒ‚ĒÄ‚Ėļ Results printed back to you
```

No intermediary. No dashboard. Just your agent and a full trading toolkit.

<summary><b>trade.py</b> ‚ÄĒ Place orders (with confirmation) + preflight dry-run</summary>

```bash
# Preflight check ‚ÄĒ verify everything before committing money (run this first)

  --type GTD --expiry 3600
```

> **Note:** Polymarket enforces a minimum 1-minute order lifetime ‚ÄĒ FOK/fill-or-kill and immediate market orders are not supported. All orders rest on the book as GTC (default) or GTD.

`--dry-run` runs 5 checks without submitting anything:

| Check | What it catches |
|---|---|
| Credentials | Bad/expired API key |
| Balance | Not enough USDC for the order size |
| Market active | Closed market or wrong token ID |
| Geoblock | `403`/`451` from Polymarket if region is blocked |
| Order signing | Key format errors (pure local crypto, no POST) |

Always shows an order preview and asks for confirmation before submitting a real order.
</details>

<details>
<summary><b>notifier.py</b> ‚ÄĒ Bot trade notifications</summary>

```bash

poly notify --bot auto_arbitrage   # filter by bot
poly notify --event trade_opened   # filter: trade_opened | trade_closed
```

All auto bots (`auto_arbitrage`, `news_trader`, `ai_automation`, `market_maker`,

a structured record to `logs/trade_notifications.json`.

Notification hooks are `try/except`-wrapped ‚ÄĒ they never crash a live bot.

Aliases: `poly notifs` ¬∑ `poly notifications` ¬∑ `poly trades`
</details>

<summary><b>geoblock.py</b> ‚ÄĒ Check if your IP is geo-blocked</summary>

Calls `GET https://polymarket.com/api/geoblock`. Returns your IP, country, region,

Status codes:

- **`close_only`** ‚ÄĒ you can close existing positions but not open new ones (PL, SG, TH, TW)
- **`blocked`** ‚ÄĒ region is restricted (AU, DE, FR, GB, US, and others)

Aliases: `poly geo` ¬∑ `poly blocked` ¬∑ `poly geo-check`

### News trading pipeline

The news system is a full 4-layer pipeline in `scripts/news/`:

```
GDELT (free, 65+ languages)  ‚ĒÄ‚Ēź
NewsAPI (optional, free key)  ‚ĒÄ‚Ē§‚ĒÄ‚Ėļ normalize ‚ĒÄ‚Ėļ cluster ‚ĒÄ‚Ėļ map ‚ĒÄ‚Ėļ score ‚ĒÄ‚Ėļ gate ‚ĒÄ‚Ėļ trade
RSS feeds (15 default)        ‚ĒÄ‚Ēė
```

| Layer | File | What it does |
|---|---|---|
| L1 Ingest | `sources/gdelt.py` | GDELT DOC 2.0 ‚ÄĒ no key, broad breaking news |
| L1 Ingest | `sources/newsapi.py` | NewsAPI.org ‚ÄĒ optional, richer metadata |
| L1 Ingest | `sources/rss.py` | 15 curated feeds (White House, Fed, Reuters, AP, SCOTUS‚Ä¶) |
| L2 Normalize | `normalize.py` | SHA-1 fingerprints, 60-domain trust table, age filter |
| L2b Cluster | `cluster.py` | Jaccard token-set clustering ‚Üí one rep per real event |
| L3 Map | `mapper.py` | Keywords ‚Üí Gamma API ‚Üí story‚ÜĒmarket relevance |
| L4 Score | `score.py` | impact = (trust ¬∑ novelty ¬∑ relevance ¬∑ specificity ¬∑ urgency)^‚Öē |
| Gate | `pipeline.py` | `edge > fees + slippage + safety_buffer` check |

**Key flags:** `--min-edge 0.06` ¬∑ `--min-impact 0.15` ¬∑ `--min-relevance 0.15` ¬∑ `--safety-buffer 0.02` ¬∑ `--max-age 60` ¬∑ `--newsapi-key KEY` ¬∑ `--skip-slippage`

<summary><b>time_decay.py</b> ‚ÄĒ Resolution-timing edge (FADE / RUSH)</summary>

```bash

```

Two sub-strategies:
- **FADE**: buy NO when market is still priced as if the event *might* happen, but the deadline is too close ‚ÄĒ exponential decay model with `DECAY_PER_DAY=0.30`
- **RUSH**: buy YES when high-probability outcome is still *under*priced near resolution

Edge formula: `fair_no = 1 - yes_price √ó (1 - 0.30)^days`.  Entry if `fair_no - live_no - fee ‚Č• min_edge`.

Enforces strict mathematical bounds between related markets:

Seven built-in logic groups (trump‚Üírepublican, btc_spot_etf‚Üíbtc_etf, NBA/NFL champions, etc.)

## Security

- Private key is loaded from `.env` at runtime and never logged or transmitted
- API key/secret undergo an entropy check at startup ‚ÄĒ placeholder strings (`YOUR_KEY`, blank values, all-same-chars, keys shorter than 32 hex chars) are rejected before any network call
- Any exception that mentions a key or secret has it redacted from output ‚ÄĒ never appears in plain text logs
- Kill switch (`poly risk kill`) is wired into all autonomous bots ‚ÄĒ a single command halts `market_maker`, `auto_arbitrage`, `ai_automation`, and `trade`
- Hard input guards (`_guards.py`) enforce a minimum $1 order size in every bot at startup; values below minimum trigger a warning, suggest a fix, and abort the run ‚ÄĒ prevents accidentally trading dust amounts
- `news_trader --interval` is clamped to a minimum of 3 minutes regardless of user input ‚ÄĒ prevents Gamma API 429 rate-limit errors from rapid polling
- Gamma API calls in the news pipeline are rate-limited to 350 ms apart ‚ÄĒ prevents burst 429 errors when many stories require market lookups in a single cycle
- `.env` is in `.gitignore` ‚ÄĒ it will never be committed
- Every trade script shows a preview and requires explicit confirmation before submitting
- Order execution uses the official [py-clob-client](https://github.com/Polymarket/py-clob-client) library only
- All read operations (market scan, research, simulation) work without credentials
